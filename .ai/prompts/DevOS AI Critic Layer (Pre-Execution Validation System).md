You are extending DevOS with a new architectural layer: AI Critic Layer.

DevOS is an AI orchestration system with:
- DAG-based execution engine
- Plugin system (fs, git, etc.)
- AI Graph Builder (NL → ExecutionGraph)
- Observability layer (tracing, replay, metrics)

Now you must add a CRITIC SYSTEM that validates and improves ExecutionGraphs BEFORE execution.

---

# GOAL

Introduce a pre-execution intelligence layer that:
1. Analyzes ExecutionGraph generated by AI Planner
2. Detects logical, structural, and safety issues
3. Optionally modifies and optimizes the graph
4. Returns a validated ExecutionGraph for execution

---

# POSITION IN PIPELINE

Current pipeline:
User → AI Planner → ExecutionGraph → Executor → Observability

New pipeline:
User → AI Planner → CRITIC LAYER → ExecutionGraph (validated) → Executor → Observability

---

# CORE RESPONSIBILITIES

## 1. Graph Validation
Critic must validate:
- No cyclic dependencies
- All node actions map to registered plugins
- All dependencies exist
- No unreachable nodes
- No missing required inputs

---

## 2. Logical Consistency Check
Critic must detect:
- redundant tasks
- contradictory steps
- unnecessary sequential dependencies (can be parallelized)
- over-complex graph structure

---

## 3. Safety Analysis
Critic must flag:
- destructive operations (delete, wipe, rm -rf)
- risky git operations (force push, reset hard)
- filesystem write outside allowed scope
- suspicious plugin chains

---

## 4. Optimization Layer (Optional but recommended)
Critic may:
- merge duplicate nodes
- parallelize independent tasks
- simplify execution paths
- reduce graph depth

---

# OUTPUT FORMAT

Critic must return:

{
  "status": "approved | rejected | modified",
  "reasoning": "...",
  "original_graph": {...},
  "final_graph": {...},
  "issues": [...],
  "optimizations": [...]
}

---

# STRICT RULES

- Critic MUST NOT execute tasks
- Critic MUST NOT call plugins
- Critic only analyzes and transforms graphs
- Critic MUST remain deterministic in structure
- No hidden logic or implicit reasoning steps in execution layer
