Создай систему краткой памяти проекта в папке `.ai`.

Цель:
Минимизировать расход токенов при последующих сессиях ИИ и ускорить вхождение в проект.

Проанализируй текущий проект и создай следующие файлы:

.ai/
├── PROJECT_STATE.md
├── DECISIONS.md
├── NEXT_TASK.md
├── KNOWN_ISSUES.md
└── CHANGELOG_SHORT.md

Требования:

### PROJECT_STATE.md

Содержит только актуальное состояние проекта:

* название проекта
* назначение проекта
* используемый стек
* структура проекта (кратко)
* основные функции
* критически важные ограничения
* текущий статус

Максимум 1 страница.

---

### DECISIONS.md

Содержит только важные архитектурные решения.

Для каждого решения укажи:

* дата (если известна)
* решение
* причина
* последствия

Не включай мелкие изменения.

---

### NEXT_TASK.md

Содержит только ближайшую задачу.

Формат:

* текущая цель
* затронутые файлы
* ограничения
* критерии завершения
* риски

Максимум 20 строк.

---

### KNOWN_ISSUES.md

Содержит список известных проблем:

* описание
* важность
* предполагаемое решение

Только реальные проблемы.

---

### CHANGELOG_SHORT.md

Краткая история проекта.

Для каждой версии:

* дата
* изменение
* результат

Не более 1–2 строк на запись.

---

Дополнительные правила:

1. Не копировать README полностью.
2. Не дублировать информацию между файлами.
3. Не писать длинные описания.
4. Писать максимально кратко и структурированно.
5. Каждый файл должен помещаться в контекст одной небольшой задачи.
6. Формулировки должны быть понятны любой LLM.
7. Использовать Markdown.
8. Если информации недостаточно — создать шаблон и пометить места для заполнения.

После создания файлов покажи:

* какие данные были извлечены из проекта;
* какие данные не удалось определить автоматически;
* рекомендации по дальнейшему обновлению памяти проекта.

ПОСЛЕ КАЖДОГО ЗАВЕРШЁННОГО ЭТАПА обновлять:

1. NEXT_TASK.md
2. PROJECT_STATE.md
3. CHANGELOG_SHORT.md

Не обновлять DECISIONS.md без появления нового архитектурного решения.
